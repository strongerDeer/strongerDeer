# 문제

도서관 대출, 구독 서비스, 구매 도서처럼 여러 곳에 흩어진 독서 기록을 한곳에 모으고, 기록을 통계와 취향 분석·추천으로 연결하고자 시작한 개인 프로젝트.

# 핵심 기능과 설계 판단

## 내 책장

**UI 판단**: 완독한 책은 표지 방향, 기록만 해둔 책은 책등 방향으로 꽂고, 인생 도서만 항상 표지를 노출.

**판단 근거**: 실제 책장에서 사람은 등록 순서가 아니라 "완독했는가·아끼는 책인가"로 책을 배치. 화면 위계도 데이터 필드 순서보다 이 인지 습관을 따를 때 더 빠르게 읽힌다고 판단.

## 독서 통계

**UI 판단**: 연도별/월별 독서량은 바 차트, 카테고리 분포는 레이더 차트로 분리하고 목표 진행률은 상단에 고정.

**판단 근거**: "얼마나"(양)와 "무엇을"(분포)은 확인하려는 질문 자체가 달라 한 차트에 담으면 둘 다 답하기 어렵다. 목표 진행률을 상단에 둔 건 통계 화면 사용자의 우선 관심사가 "올해 목표 달성도"라고 봤기 때문.

## AI 취향 분석

**UI 판단**: 완독·별점 데이터를 OpenAI API로 분석해 독서 성향 요약과 추천 도서를 함께 보여주고, 분석 결과는 저장해 재조회 가능하게 구성.

**판단 근거**: 매번 새로 분석하면 대기 시간이 반복되고 같은 데이터도 결과가 미묘하게 달라짐. 분석 시점 결과를 저장하면 사용자가 이전과 현재를 비교하며 취향 변화를 확인할 수 있다고 판단.

# 주요 기술 결정

- Next.js App Router에서 랜딩 랭킹과 대시보드 캘린더 영역을 Suspense와 ErrorBoundary로 분리해, 일부 영역이 실패해도 주요 화면을 유지하도록 구성
- 알라딘 도서 API를 서버에서 중계하고 1시간 캐시를 적용해 외부 API 호출량과 브라우저 보안 제약을 축소
- 탐색 방식을 화면 목적에 맞게 구분 — 팔로잉 활동 피드는 무한 스크롤로 자연스러운 탐색을, 특정 책을 찾는 내 책 리스트는 페이지네이션으로 목적성 있는 탐색을 지원
